# Finance-Dashboard-Backend

Backend assignment code lives in **`finance-backend/`**. This repository root also contains **`render.yaml`** so you can deploy the API to [Render](https://render.com) in a few clicks.

---

## Go live on Render (step by step)

I cannot log into your accounts for you. Do these on **your** machine and in **your** browser.

### 1. Push this repo to GitHub

```bash
cd ~/Desktop/Project
git add .
git commit -m "Deploy: Render blueprint + production listen host"
git push origin main
```

### 2. MongoDB Atlas

- Cluster **must be running** (if paused, click **Resume**).
- **Network Access:** add **`0.0.0.0/0`** so Render can connect (or tighten later).
- **Database Access:** note username + password.
- **Connect** → Drivers → copy SRV string and insert **`/finance_dashboard`** before `?`, e.g.  
  `mongodb+srv://USER:PASSWORD@cluster....mongodb.net/finance_dashboard?retryWrites=true&w=majority`  
- **Never commit** this URI; paste it only in Render (and local `.env`).

### 3. Create the web service on Render

1. [dashboard.render.com](https://dashboard.render.com) → **New +** → **Blueprint**.
2. Connect the **same GitHub repo** as this project.
3. Render should detect **`render.yaml`** at the repo root (`rootDir: finance-backend`).
4. When prompted, set **`MONGODB_URI`** to your full Atlas string (with `/finance_dashboard`).
5. Deploy and wait until status is **Live**.

`MOCK_JWT_SECRET` is auto-generated by the blueprint; you can leave it or replace it in **Environment**.

### 4. Seed demo users (once per database)

Render → your service → **Shell**:

```bash
node src/scripts/seed.js
```

### 5. Test your public URL

Replace `https://YOUR-SERVICE.onrender.com` with the URL Render shows:

```bash
curl -s https://YOUR-SERVICE.onrender.com/health
curl -s -X POST https://YOUR-SERVICE.onrender.com/api/v1/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"admin@seed.local"}'
```

- **Swagger:** `https://YOUR-SERVICE.onrender.com/api-docs`
- **OpenAPI JSON:** `https://YOUR-SERVICE.onrender.com/openapi.json`
